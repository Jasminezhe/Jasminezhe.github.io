HTML5语义化标签

Veröffentlicht am 2016-06-27

HTML5语义化标签

1、新增标签

（1）header

header标签定义文档的页眉（介绍信息）。

（2）nav

nav标签定义导航链接的部分。

（3）aside

aside标签定义 article 以外的内容。aside 的内容应该与 article 的内容相关。

（4）footer

  • footer标签定义文档或节的页脚；

  • footer元素应当含有其包含元素的信息。页脚通常包含文档的作者、版权信息、使用条款链接、联系信息等等。

（5）section

section标签定义文档中的节（section、区段）。比如章节、页眉、页脚或文档中的其他部分。是语义的div。

（6）article

article标签定义外部的内容。可以用来定义文档内容。更加语义化的div。

（7）hgroup

hgroup标签用于对网页中的标题进行组合。

<hgroup>
    <h1></h1>
    <h2></h2>
</hgroup>

（8）details

details元素用来描述文档或文档片段的信息，summary元素与details元素一同使用，用于说明文档的标题，同时，summary元素应为details元素的第一个子元素。

<details>
    <summary>我是一级目录</summary>
    <details>
        <summary>我是二级目录</summary>
            <details>
                <summary>我是三级目录</summary>
            </details>
    </details>
</details>

（9）dialog

dialog标签定义对话，比如交谈。默认情况下不显示，需要定义open属性。

（10）figure

figure标签用于定义图标元素，可以搭配figcaption标签使用，figcaption标签可以用来定义图标标题。

（11）address

  • 如果 address 元素位于 body 元素内部，则它表示该文档作者/所有者的联系信息；

  • 如果 address 元素位于 article 元素内部，则它表示该文章作者/所有者的联系信息；

  • 不应该使用
    标签来描述邮政地址，除非这些信息是联系信息的组成部分；

  • 元素通常被包含在footer元素的其他信息中。

2、短语句

（1）mark

mark标签常用于标记高亮文本。

（2）meter

  • 定义度量衡。表示在一定范围内的值，如投票占比、使用量等;

  • value:实际度量的值，默认为0，可以为整数或者浮点数;

  • high、max:范围的上限值、最大值：默认是1;

  • low、min:范围的下限值、最小值:默认是0;

  • optimum:最佳的值，必须在min属性值与max属性值之间

（3）progress

进度条。属性有value、max、min。

（4）time

定义时间和日期。

（5）wbr

定义换行。与br不同，br是强制换行，wbr是当宽度不够时进行换行，并且wbr是双标签，换行时wbr内的所有内容都一起换行。

（6）abbr

定义缩写。

（7）cite

引入。字体为斜体。

（8）blockquote

引入。有默认的margin值，一般用于短文本。

（9）q

引入。有引号，一般用于长文本。

（10）output

output 标签定义不同类型的输出，比如脚本的输出。

（11）pre、code

pre用来标记多行代码，code用来标记单行代码。

2、表单元素

（1）input

  • tel：用于输入电话号码；

  • search：用于提示用户输入要搜索的文字，与text的区别在于样式上；

  • url：用于输入单个网址；

  • email：用于输入电子邮件地址；

  • datetime：用于输入时区设置为UTC的日期和时间；

  • date：用于输入不含时区的日期；

  • month：用于输入含年份和月份但不含时区的日期；

  • week:用于输入含年份和月份但不含时区的日期；

  • time：用于输入含小时、分钟、秒和秒的小数部分但不含时区的时间值；

  • datetime-local：用于输入不含时区的日期和时间；

  • number：用于输入数字；

  • range：用于输入数字，但与number的区别在于无需输入具体数字；在大部分支持该类型的浏览器中，范围控件的实施形式为滑块；

  • color：用于通过颜色池控件选择颜色；

###（2）label

  • label 标签为 input 元素定义标注（标记）。

  • label 元素不会向用户呈现任何特殊效果。不过，它为鼠标用户改进了可用性。如果您在 label 元素内点击文本，就会触发此控件。就是说，当用户选择该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上。

3、表单属性

（1）autofocus

用于网页加载后对焦到相关元素上的输入；autofocus的目标可以是输入、选择、文本区域和按钮；

（2）placeholder

  • 用于提示用户应输入的数据类型；

  • 在对焦到相关元素以及用户输入数据之前，系统会以浅色文字显示占位符值。您可以在输入和文本区域中指定该值。

（3）required

如果使用该属性，则字段是必填（或必选）的；

4、音视频

(1)audio、video

  • audio标签用来定义音频文件；video标签用于定义视频文件；

  • autoplay：控制文件自动播放；

  • controls：打开控制面板；

  • lopp：循环播放；

  • preload：视频、音频提前加载；

  • src：文件网址或者本地路径；

（2）track

定义用在媒体播放器中的文本轨道；如歌词文件或字幕文件；

（3）source

  • 有title属性，是单标签；

  • 可以嵌套在audio和video标签中使用，为一个文件添加多种格式，以防浏览器不兼容。

响应式布局(一)--meta标签

Veröffentlicht am 2016-06-24

响应式布局(一)–meta标签

meta提供关于HTML文档的元数据。元数据不会显示在页面上，但是对于机器是可读的。它可用于浏览器（如何显示内容或重新加载页面），搜索引擎（关键词），或其他 web 服务。

1、申明编码

<meta charset='utf-8' />

charset 属性是 HTML5 中的新属性，规定在外部脚本文件中使用的字符编码。如果外部文件中的字符编码与主文件中的编码方式不同，就要用到 charset 属性。默认的字符编码是 ISO-8859-1。

2、浏览器

(1)优先使用 IE 最新版本和 Chrome

<meta http-equiv="X-UA-Compatible"                       content="IE=edge,chrome=1" />

(2)内核控制

国内浏览器很多都是双内核（webkit和Trident），webkit内核高速浏览，IE内核兼容网页和旧版网站。而添加meta标签的网站可以控制浏览器选择何种内核渲染。
<meta name="renderer" content="webkit|ie-comp|ie-stand">

(3)国内双核浏览器默认内核模式

  • 搜狗高速浏览器、QQ浏览器：IE内核（兼容模式）
  • 360极速浏览器、遨游浏览器：Webkit内核（极速模式）

3、页面关键字

<meta name="keywords" content="your tags，” />

每个网页应具有描述该网页内容的一组唯一的关键字。 使用人们可能会搜索，并准确描述网页上所提供信息的描述性和代表性关键字及短语。标记内容太短，则搜索引擎可能不会认为这些内容相关。关键词最好不要超过12个。

4、页面描述

<meta name=“description"content="120｜ 150 words" />
每个网页都应有一个不超过 150 个字符且能准确反映网页内容的描述标签。

5、搜索引擎索引方式

<meta name=“robots” content="index,follow" />

搜索引擎索引方式，robotterms是一组使用逗号(,)分割的值，通常有如下几种取值：none，noindex，nofollow，all，index和follow。确保正确使用nofollow和noindex属性值。

  • all：文件将被检索，且页面上的链接可以被查询；
  • none：文件将不被检索，且页面上的链接不可以被查询；
  • index：文件将被检索；
  • follow：页面上的链接可以被查询；
  • noindex：文件将不被检索；
  • nofollow：页面上的链接不可以被查询。

6、页面重定向刷新

<meta http-equiv="refresh" content="0;url=" />

content内的数字代表时间（秒），既多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

7、网页作者信息

<meta name="author" content="author name" />

8、CSS3浏览器前缀

  • 火狐Firefox: -moz-

  • 苹果Safari: -webkit-

  • 谷歌Chrome: -webkit-

  • Opera: -o-

  • IE: -ms-

单位

Veröffentlicht am 2016-06-23

1、px

PX是Pixel的缩写，也就是说像素是指基本原色素及其灰度的基本编码，由 Picture(图像) 和 Element（元素）这两个单词的字母所组成的。可以用来设置距离、宽高、文字大小等。浏览器默认文字大小是16px。

2、em

em不是一个固定值，是一个相对单位，会继承父级的字体大小，浏览器默认字体大小是16px。例：

3、rem

rem也是一个相对单位,r代表“root”,是相对html文件的根元素即html。通过它既可以做到只修改根元素就成比例地调整所有字体大小，又可以避免字体大小逐层复合的连锁反应。例：

4、min-width、min-height、max-width、max-height

min-width和min-height，max-width和max-height后面一般加具体像素值，常用于限制图片尺寸大小。例如,用div插入很多图片的时候，图片尺寸可能不一样，那么撑起盒子的宽高也不一样，通常会造成图片混乱。

5、百分比

设置宽高以百分比计算，随外级对象宽度变化而自适应按百分比显示宽度，实现响应式布局。font-size也可以用百分比设置，并且是继承父级文字大小。

6、vh和vw

在进行响应式布局时，我们常常会使用百分比来布局，然而CSS的百分比不总是解决每个问题的最佳方案，CSS的宽度相对于离它最近的父元素的宽度。 如果你想使用视口的宽度、高度而不是父元素的宽高，可以使用vh和vw单位。 1vh = viewportHeight 1/100; 1vw = viewportWidth 1/100; 使用vh、vw就可以保证元素的宽高适应不同设备。

HEXO博客搭建

1、配置nodeJS

  • 在https://nodejs.org/en/网站下载合适版本的nodeJS；

  • 测试是否安装成功，在终端命令下输入node -v回车，会出现版本信息，则安装成功。

2、安装HEXO

  • 终端命令下输入npm install hexo -g安装HEXO，如果提示错误信息root Administrator,则重新输入sudo npm install hexo -g以管理员方式安装，提示输入密码；

  • 测试是否安装成功，在终端命令下输入hexo,出现以下命令行信息，则安装成功；

  • 在终端下进入桌面，使用mkdir命令创建一个文件夹blog，并进入该文件夹；

  • 输入命令hexo init进行hexo目录的初始化设置；

  • 设置完成后输入命令hexo server启动本地服务器，启动成功后，会提示本地服务器地址，在浏览器中打开后可以本地预览博客。如下图。

3、博客编辑

  • 在终端下进入blog文件夹，输入命令hexo new 文件名，可以新建一个新的博客；

  • 使用MOU软件写入博客内容。

4、博客互联网发布

(1)首先需要github账号，可以在官网注册

(2)配置ssh-key

  • 终端命令下输入ssh-keygen，连续三次回车；

  • 输入vim ~/.ssh/id_rsa.pub，会出现ssh-key的信息，复制下来；

(3)配置github

  • 选择settings进入后在左标签中选择SSH and GPG keys;


  • 选择配置new ssh key，然后在key中粘贴刚刚复制下来的ssh-keygen信息，title随便写。


(4)配置github与本地机器验证

  • 终端下输入git config –global user.name “github用户名”；

  • 终端下输入git config –global user.email “github注册时的邮箱地址”；

没有提示异常就OK啦~~

(5)新建博客域名仓库

  • 选择新建仓库，输入仓库域名，注意，一定要是用户名.github.io,然后选择public，勾选新建readme文档，点击创建；


(6)配置config.yml文件

  • 配置deploy，type: git，repo:域名仓库的https地址，在仓库中点击clone or download 复制仓库域名地址，并粘贴到repo中；


    (7)发布博客

  • 在终端命令中输入npm install hexo-deployer-git –save下载安装配置工具；

  • 可以在本地服务器下预览博客，如果没有问题在终端下输入hexo g将blog中的.md文件编译成css、js等文件；

  • 输入hexo d进行发布，第一次发布会要求输入github的用户名和密码，发布成功后的信息如下。

最后，在浏览器中输入地址，即用户名.github.io就可以在网上查看博客了~~~~
